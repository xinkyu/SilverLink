# SilverLink 银龄守护：深度设计方案

- **交互原则**：**Zero-UI (零界面)** 优先。老人不需要学习如何点击菜单，一切通过自然语言和被动感知完成。

---

## 详细功能模块设计 (Functional Design)

### 1. 全双工情感陪伴系统 (The Empathic Companion)

这是App的核心灵魂，基于`Gemma 2B` + `SenseVoice` + `Kokoro-82M` 实现。

- **功能点 1.1：情绪镜像与抚慰 (Emotional Mirroring)** 已实现
  - **逻辑**：系统不仅识别文本，还通过`SenseVoice`分析语调（Pitch/Energy）。
  - **场景**：
    - 老人叹气说：“哎，今天腿又疼了。”（识别情感：Negative/Pain）
    - SilverLink回应（调整`Kokoro` TTS参数，语速放慢，语调温柔）：“奶奶，是不是变天了膝盖不舒服？要不要我帮您打开电热毯的开关？”（结合智能家居控制）
  - **技术细节**：维护一个全局变量 `current_mood_state`，动态调整Prompt的前缀，例如 `System Prompt: "You are a caring grandson. The user sounds sad. Be gentle."`

- **功能点 1.2：方言自适应 (Dialect Adaptation)**
  - **痛点**：很多老人只会说方言。
  - **实现**：利用`Sherpa-ONNX`的多语言/方言模型库（如支持普通话与部分方言混合识别），并在Prompt中指示Gemma用简单的口语回答，而非书面语。

- **功能点 1.3：主动式闲聊 (Proactive Interaction)**
  - **逻辑**：利用Android `SensorManager`（计步器、光线感应）。
  - **场景**：检测到老人长时间（>3小时）未移动手机且环境光亮。
  - **动作**：主动唤醒屏幕并说话：“爷爷，您坐很久了，要不要起来喝杯水，或者我们听段京剧？”

### 2. 多模态记忆回溯 (The Memory Time-Machine)

基于端侧RAG (`ObjectBox` + `CLIP`) 实现。

- **功能点 2.1：老照片的故事 (Photo Talk)**
  - **流程**：后台静默扫描相册 -> CLIP生成向量 -> 存入ObjectBox。
  - **交互**：老人点开一张旧照片，或者直接问“那年去北京是谁带我去的？”
  - **响应**：系统检索相关照片的元数据（时间、地点）和图像内容，合成回答：“是2018年，您的儿子带您去的故宫，照片里您笑得很开心，穿着红色的羽绒服。”

- **功能点 2.2：数字遗忘胶囊 (Digital Amnesia Defense)**
  - **功能**：针对阿尔茨海默症早期。系统会定期“考考”老人。
  - **场景**：展示一张家人的照片，“奶奶，您还记得这是谁吗？”
  - **反馈**：如果老人答不上来，系统温柔提醒并记录“认知衰退指数”，生成周报发送给子女。

### 3. 游戏化健康管理 (Gamified Health Guard)

- **功能点 3.1：孙辈语音激励 (Voice Cloning Incentive)**
  - **创新**：不使用机械闹钟声。
  - **实现**：预先录制几句孙子/孙女的声音，或者使用Few-shot TTS克隆子女声音（需授权）。
  - **场景**：吃药时间到，播放声音：“爷爷，快吃降压药，吃完了周末带我去游乐园哦！”

- **功能点 3.2：药盒视觉核对 (Visual Pill Check)**
  - **交互**：老人拿起药瓶问：“是吃这个吗？”
  - **技术**：调用摄像头 -> MobileVLM / YOLO -> 识别药瓶标签与处方库比对。
  - **回答**：“不对哦爷爷，这是阿司匹林，现在该吃的是那个白瓶子的钙片。”

---

这是一份为**SilverLink 银龄守护**量身定制的深度UI/UX设计方案。

设计核心原则：**去App化（De-Appification）**。
对于老年用户，我们不设计“菜单”、“设置”或“下拉刷新”。整个界面应该感觉像是一个**有生命的窗口**，而非一个复杂的工具软件。

## 🎨 视觉设计规范 (Visual System)

- **色调 (Color Palette)**:
  - _主色 (Warmth)_: **#FFAB91 (暖杏色)** - 用于背景基调，传递温暖。
  - _强调色 (Action)_: **#FF7043 (深橘红)** - 用于核心交互按钮/光球。
  - _安全色 (Safe)_: **#66BB6A (柔和绿)** - 用于确认、健康状态正常。
  - _警示色 (Alert)_: **#EF5350 (清晰红)** - 仅用于吃药提醒和SOS。
  - _文字色_: **#37474F (深蓝灰)** - 避免纯黑，减少对老人眼睛的刺痛，对比度>7:1。
- **字体 (Typography)**:
  - 使用系统默认无衬线字体（Roboto/HarmonyOS Sans），字重加粗。
  - **最小字号**: 24sp (正文)，36sp (标题/关键信息)。
- **图标 (Iconography)**:
  - **拟物化 (Skeuomorphism)**：避免抽象的线条图标。药瓶就是真实的药瓶照片，电话就是一个老式话筒的图样。

## 一、 注册与初始化 (The "Invisible" Setup)

**策略**：为了降低门槛，**老人端无需注册**。所有配置由**子女端**完成，老人只需扫码或输入简短配对码。

### 1.1 启动页 (Splash Screen) 已实现

- **画面**：纯净的暖杏色背景，中心是一个简笔画的“两只手相握”的Logo。
- **文字**：淡入显示“SilverLink · 银龄守护”。
- **语音**：自动播放一声温柔的问候：“欢迎回家”。

### 1.2 身份选择页 (Role Selection)已实现

- **布局**：屏幕上下均分两个巨大卡片。
- **上卡片**：👴 **我是长辈** (配图：一位慈祥的老人)。
- **下卡片**：👩‍💼 **我是家人** (配图：年轻子女)。

### 1.3 家人端：配置流程 (The Setup Wizard) 

_子女在自己手机上操作，为父母配置“数字伴侣”。_

1.  **基础信息录入**：输入老人的称呼（如“王爷爷”）、家乡（用于方言模型加载）。已实现
2.  **声音克隆 (Voice Cloning)**：
    - **UI**：屏幕显示一段文字：“爷爷，记得按时吃药哦”。
    - **操作**：按住录音按钮朗读。系统提示“正在生成您的AI语音包...”。
3.  **记忆植入 (Memory Injection)**：
    - **UI**：类似微信发图。批量上传老照片，并为每张照片录入一句话描述（构建RAG数据库）。
4.  **生成配对码**：屏幕显示一个巨大的6位数字（如 `882 109`）和一个二维码。已实现

### 1.4 长辈端：激活 (Activation)已实现

_老人点击“我是长辈”后进入。_

- **UI**：屏幕中央一个巨大的输入框，数字键盘自动弹出。
- **引导**：语音播报“请让您的孩子输入这几个数字”。
- **成功后**：屏幕炸开礼花特效，光球出现，用子女的声音说：“爷爷，我住进你的手机啦！”

## 二、 核心主界面：无界伴侣 (The Living Orb)

这是老人90%时间面对的界面。没有列表，没有Tab栏。

### 2.1 待机/陪伴状态 (Idle State)

- **背景**：根据当地时间动态变化的渐变色（清晨是淡黄，夜晚是深蓝）。
- **视觉中心**：**呼吸光球 (The Orb)**。
  - 直径约占屏幕宽度 1/2。
  - 材质感：像毛玻璃或发光的棉花糖，缓慢膨胀收缩（模拟呼吸频率）。
- **辅助信息**：
  - 左上角：超大字体显示 **14:30**。
  - 右上角：**天气图标 + 温度**。
  - 底部：一个实体的、拟物化的红色 **“呼救 (SOS)”** 拉环/按钮（需长按3秒触发，防止误触）。

### 2.2 全双工交互状态 (Interaction State)已实现

_当老人开口说话，或点击光球时。_

- **聆听中 (Listening)**：
  - 光球变为暖橙色，周围出现声波纹路（Visualizer），随老人音量跳动，给予即时视觉反馈。
- **思考中 (Thinking)**：
  - 光球快速旋转，颜色在橙色和青色间流转（代表AI在检索/推理）。
- **说话中 (Speaking)**：
  - 光球变形为声波律动。
  - **字幕层**：屏幕下方实时显示AI说的话（超大字号，卡拉OK式逐字高亮），辅助听力下降的老人。

### 2.3 情感镜像反馈 (Emotional Mirroring UI)

_对应功能点 1.1_

- **场景**：当`SenseVoice`检测到老人语调悲伤。
- **UI变化**：
  - 背景色温瞬间降低，变为柔和的莫兰迪色。
  - 光球的边缘变得模糊、柔软。
  - 光球上方浮现一个半透明的“拥抱”Emoji或心形符号。

## 三、 功能场景UI设计

### 3.1 记忆时光机 (Memory Time-Machine)

_对应功能点 2.1 & 2.2_

**触发方式**：语音指令“看看照片”或“我想孙子了”。

- **布局**：**沉浸式横向画廊 (Immersive Gallery)**。
- **视觉**：
  - 全屏显示一张老照片，背景做高斯模糊处理。
  - **照片动效**：利用Ken Burns Effect（缓慢推拉镜头），让静止照片动起来。
- **交互**：
  - **“讲故事”模式**：照片下方不显示文件名，而是显示AI生成的叙述：“这是2018年春节，大家在老家门口贴对联。”
  - **悬浮麦克风**：右下角始终悬浮麦克风。老人问：“这是哪一年？” AI回答后，文字气泡浮在照片上方。
- **数字遗忘胶囊 (Quiz Mode)**：
  - **UI**：照片四周出现柔和的金边框。
  - **提示文案**：屏幕上方显示“猜猜他是谁？”
  - **反馈**：老人回答正确，屏幕撒花，播放欢快的音效；回答错误，光球温柔地说“没关系，这是您的侄子小刚”。

### 3.2 游戏化服药提醒 (Gamified Medication)已实现

_对应功能点 3.1_

**触发方式**：闹钟时间到（全屏强制弹窗，不可被后台杀掉）。

- **视觉**：
  - 背景变为醒目的**橘红色条纹**（警示但不过分惊悚）。
  - 中央：**高清药物实拍图**（例如一粒白色的药片放在手心里）。
  - 文字：**“吃一片 降压药”**（数字特大）。
- **听觉**：
  - 播放子女克隆语音：“爷爷，快吃药啦，吃完身体棒！”
- **操作按钮**：
  - 左侧（绿色实体大按钮）：**✅ 我吃完了**（点击后播放孙子声音：“爷爷真棒！”）。
  - 右侧（灰色小按钮）：**稍后提醒**（5分钟后再次触发）。

### 3.3 药盒视觉核对 (Visual Pill Check)

_对应功能点 3.2_

**触发方式**：老人说“帮我看看这个药对不对” -> 自动调用摄像头。

- **布局**：**极简取景器**。
- **视觉元素**：
  - 全屏相机预览。
  - 中央有一个白色的虚线框，提示老人把药瓶放进去。
  - **手电筒开关**：如果环境光暗，自动高亮显示“打开手电筒”的大按钮。
- **识别反馈**：
  - **正确 (Match)**：虚线框变**实心绿色**，弹出巨大的绿色对勾 ✅。语音：“没错，就是这个阿司匹林。”
  - **错误 (Mismatch)**：虚线框变**红色震动**，弹出红色叉号 ❌。语音急促提醒：“不对！爷爷，这是钙片，不要吃！”

### 3.4 主动式闲聊唤醒 (Proactive Interaction)

_对应功能点 1.3_

**触发方式**：手机黑屏放在桌上，传感器检测到久坐。

- **UI流转**：
  1.  **黑屏 -> 微光**：屏幕并非突然亮起，而是像日出一样缓慢变亮（Fade-in duration: 2s）。
  2.  **光球跳出**：光球从屏幕底部探出半个头，像个调皮的孩子。
  3.  **气泡提示**：光球上方弹出一个气泡：“爷爷，该活动啦！”
  4.  **语音介入**：如果老人没反应，AI才开始说话。

## 💡 比赛加分设计细节 (UX Micro-interactions)

1.  **防误触 (Anti-Mistouch)**：
    - 所有关键操作（如删除、退出、SOS）不使用点击，而使用**“长按并填满圆环”**的交互。即用户按住按钮，一个进度条绕着手指转一圈，转满才生效。
2.  **音量自适应**：
    - 当AI检测到老人的声音很小或背景很吵时，自动提高TTS的输出音量，并用大字提示“我大点声说”。
3.  **拟人化充电**：
    - 当老人给手机充电时，光球会变成一个“吃东西”或“睡觉”的表情，说：“我也要补充能量啦，爷爷早点睡。”

这套UI设计方案完全剥离了传统软件的冷冰冰感，将**情感计算**和**多模态技术**封装在了一个温暖、直观的交互壳中，非常契合大赛“AI无界·人文关怀”的主题。

---

在演示Demo或撰写文档时，强调以下细节可以大幅加分：

1.  **离线优先演示**：在演示现场，直接**开启飞行模式**。展示SilverLink依然能流畅对话、检索本地照片。这强力证明了“端侧智能”和“隐私保护”。
2.  **情感反馈闭环**：在演示中特意对着手机发火或哭泣，展示UI颜色变暖，TTS语调变柔和。这是目前大多数商业App做不到的。
3.  **人文关怀细节**：UI中设计一个**“防误触机制”**。例如，所有重要操作（如删除、紧急呼叫）都需要长按3秒，并伴有圆环进度条提示，防止老人手抖误操作。

这是一个非常有趣且富有张力的组合。

**“SilverLink 银龄守护”** 是一个充满人文关怀、温暖、柔和的产品。
**“软件硬做”** 是一个听起来非常硬核、极客、有一种“死磕技术”精神的团队名。

这两个Logo放在一起，恰好形成了一种**“铁汉柔情”**的反差感：团队拥有硬核的技术实力（硬做），只为给老人提供最温柔的守护（银龄）。

### 第一部分：产品 Logo 设计 —— “SilverLink 银龄守护”

**设计核心词**：连接、温暖、呵护、无穷大（∞）、S字母。
**应用场景**：手机App图标、启动页中心。

#### 方案 A：【温情纽带】(The Warm Knot) —— _推荐_

这是最符合App UI风格的设计，极简且语义明确。

- **图形描述**：
  - 主体是一个抽象的 **“S”** 形状（代表 Silver）。
  - 这个“S”不是普通的线条，而是由两条**柔和的丝带**交织而成。
  - **上半部分**是银灰色（#B0BEC5），代表科技与理性，也象征“银发”。
  - **下半部分**渐变为暖橙色（#FF7043），代表关怀与温度。
  - 两条丝带在中心交汇处形成了一个**隐喻的爱心形状**。
- **背景**：纯白底色，或者微圆角矩形（App图标形状）。
- **含义**：科技（银色）向下延伸，化作温暖（橙色）托举住老人。

### 第二部分：团队 Logo 设计 —— “软件硬做”

**团队名解读**：“软件硬做”通常意味着“把软件当硬件一样严谨地做（高质量）”或者“用软件解决硬件问题（边缘计算/端侧模型）”。
**设计核心词**：硬核、结构化、像素风、工业感、代码。

#### 方案 A：【代码铁砧】(The Code Anvil) —— _最符合队名气质_

体现“千锤百炼”的工匠精神。

- **图形描述**：
  - 主体是一个黑色的、厚重的**铁砧**（Anvil，打铁用的底座）剪影。
  - 在铁砧的侧面，镂空刻着一行绿色的**代码符号** `</>` 或 `{ }`。
  - 或者，铁砧上方悬浮着一把像素风格的**锤子**，正在敲打一个发光的**Bug**。
- **配色**：黑（Black）、深灰（Dark Grey）、荧光绿（Terminal Green）。
- **字体**：使用粗壮的**黑体**或**像素字体**书写“软件硬做”。
- **含义**：我们的软件不仅是代码，更是经过千锤百炼的工业级产品；不管是多难的算法（硬骨头），我们都啃得下来。
