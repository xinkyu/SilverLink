# 技术创新点说明（建议稿）

## 建议优先采用的两项核心创新点
1. **端侧实时对话的可打断语音链路（VAD + 回声抑制 + 打断控制）**：该方案直接决定实时对话体验与稳定性，是产品的核心交互底座，创新度与工程难度最高。
2. **声音复刻的端云直传链路（绕过云函数限制）**：该方案解决了训练素材上传的工程瓶颈，保障声音复刻闭环可用，是语音个性化能力落地的关键支撑。

---

## 1. 端侧实时对话的“可打断”语音链路（核心）
**技术要点**：在端侧构建“录音采集 → VAD → 实时识别 → 播放 → 打断”的闭环，结合多重回声抑制策略确保低误触发和低延迟。

**实现细节**：
- **端侧 VAD**：使用 WebRTC VAD 的 VERY_AGGRESSIVE 模式并结合帧长度与静默阈值，增强语音起止判定准确度，减少静音/环境噪声误触发。
- **回声抑制与打断保护**：在播放期间引入更高 RMS 阈值、连续多帧确认及冷却期，确保 AI 播放语音不会反向触发识别，同时用户可在有效语音能量下实现打断。
- **端侧状态机**：将 Listening / Speaking / Processing / Interrupted 状态明确分层，保障打断、回放、识别之间的切换有序、可解释。

**创新性说明**：
将“VAD + 端侧回声抑制策略 + 播放冷却期 + 连续帧确认”组合为可打断链路，兼顾低延迟与抗回声误触发，在移动端实时对话中具有较高工程创新度。

---

## 2. 声音复刻的端云直传链路（核心）
**技术要点**：将声音复刻训练音频上传改为端侧直传云存储，通过云端签名授权与直链 URL/代理下载并行方案，规避云函数大小限制并提升第三方接口兼容性。

**实现细节**：
- **直传 COS**：由云函数下发上传凭证（PUT 签名），客户端直接上传音频到 COS，避免云函数 6MB 上传限制。
- **双 URL 策略**：优先使用不含签名参数的 COS 直链 URL（便于第三方模型训练接口识别），当直链不可用时自动回退代理下载 URL。
- **端云分工清晰**：云端负责授权与安全管控，客户端负责大文件上传与回调处理，降低整体失败率。

**创新性说明**：
以“端侧直传 + 直链/代理双通道”解决语音复刻训练素材上传的限制与兼容问题，属于工程架构层面的有效创新，直接提升核心能力可用性。

## 3. 端侧硬件回声消除的共享会话方案
**技术要点**：让录音与播放共享同一 audioSessionId，使 AcousticEchoCanceler（AEC）能够识别播放音频并在麦克风输入中消除回声。

**实现细节**：
- 录音端创建 `AudioRecord` 后将其 sessionId 写入共享管理器。
- 播放端通过 `AudioTrack` 复用同一 sessionId，并在播放前将 MP3 解码为 PCM 以保证音频链路可控。
- 同时启用 AEC/NS/AGC（回声消除/降噪/自动增益），提高人声清晰度。

**创新性说明**：
将硬件级回声消除能力与端侧实时对话耦合，通过共享会话解决“播放音频回灌导致误识别”的行业常见问题。

---

## 4. 低延迟流式 ASR 的手动分段控制
**技术要点**：通过 WebSocket 流式上传 PCM，并手动控制 append / commit / finish，精准管理识别边界与延迟。

**实现细节**：
- 音频帧经 VAD 过滤后进入上传通道，仅在语音有效时发送。
- 使用 commit + finish 作为一次识别的收束机制，避免长段语音带来的尾部延迟。
- 支持 partial 回传，界面可同步展示“正在听”。

**创新性说明**：
以“端侧 VAD + 手动分段控制”的组合，优化流式 ASR 的端侧延迟与响应速度，提升实时交互体验。

---

## 5. 流式 TTS 双工协议接入与音频缓冲拼接
**技术要点**：采用 WebSocket duplex 方式进行语音合成，按块接收音频并在端侧拼接，实现边合成边播放的低延迟体验。

**实现细节**：
- run-task 发送合成请求后立即进入数据流接收。
- 二进制音频帧实时写入缓冲区，结束时统一封装成可播放数据。
- 针对方言与音色切换提供统一指令构建逻辑。

**创新性说明**：
以流式协议替代传统 HTTP 大包下载，提高语音响应速度，适配实时对话场景。

---

## 6. 跌倒检测的“规则 + 轻量特征分类”融合
**技术要点**：在传统规则检测基础上叠加轻量特征分类器，用于识别软着陆与复杂场景，提高鲁棒性。

**实现细节**：
- 规则链路：自由落体 → 撞击 → 静止的三阶段阈值与时间窗。
- 轻量分类：基于幅度范围、标准差、自由落体样本数等特征计算跌倒概率。
- 通过确认期与冷却期降低误报。

**创新性说明**：
将“多阶段规则 + 特征评分”融合在端侧轻量实现，兼顾准确率与运行成本。

---

## 7. 多传感器冗余的久坐监测策略（备选）
**技术要点**：用计步器作为主通道、加速度计作为备用通道，提升对传感器缺失或异常的容错能力。

**实现细节**：
- 计步器优先判断真实活动变化。
- 加速度计作为补偿通道，提供低成本的活动检测。

**创新性说明**：
以多传感器冗余提升稳定性，属于工程可靠性创新点。
